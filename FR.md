<ΛΑ-1>

Ο χρήστης πρέπει να μπορεί να δημοσιεύει νέα λίστα αγαπημένων.

Περιγραφή: Κάθε χρήστης θα πρέπει να έχει τη δυνατότητα δημιουργίας και δημοσίευσης νέας λίστας 
οποιασδήποτε κατηγορίας επιθυμεί. Έτσι θα μπορεί να δημιουργεί λίστες ταινιών, μουσικής και βιβλίων
και να τις δημοσιεύει ώστε να μπορούν να τις βλέπουν και οι υπόλοιποι χρήστες.

User priority (5/5): Η παραπάνω λειτουργία είναι πολύ σημαντική για το χρήστη αφού χωρίς αυτή 
οι λίστες που θα δημιουργεί δε θα είναι ορατές από τους υπόλοιπους χρήστες και δε θα μπορούν να βαθμολογηθούν
και να ανέβουν στην κατηγορία κατάταξης.

Technical priority (5/5): Η <ΛΑ-1> είναι από τις βασικές λειτουργίες της εφαρμογής οπότε 
η συγκεκριμένη υλοποίηση κρίνεται απαραίτητη για την σωστή χρήση της.

Stability (4/5): Αποτελεί πολύ σημαντική λειτουργία της εφαρμογής κι έτσι είναι χρήσιμο ο τρόπος 
με τον οποίο ο χρήστης δημοσιεύει λίστες να είναι συγκεκριμένος.

<ΛΑ-2>

Ο χρήστης πρέπει να μπορεί να δει τις λίστες των άλλων χρηστών.

Περιγραφή: Αρχικά ο χρήστης θα βλέπει στην αρχική του σελίδα τις δημοσιευμένες λίστες με βάση
τα κριτήρια που έχουν προκαθοριστεί (βαθμολογία, πρόσφατα δημοσιευμένες κλπ). Στη συνέχεια θα πρέπει
να έχει τη δυνατότητα να επιλέγει κάποια από αυτές τις λίστες και να βλέπει το περιεγχόμενό τους. 
Όταν θα βρίσκεται στην σελίδα της εκάστοτε λίστας θα του εμφανίζονται και οι επιπλέον
επιλογές σχετικά με αυτές (βαθμολόγηση λίστας,σχόλια κλπ). 

User priority (5/5): Η δυνατότητα αυτή είναι πολύ σημαντικό να παρέχεται στο χρήστη αφού για να μπορέσει 
να βαθμολογήσει ή να σχολιάσει μια λίστα άλλου χρήστη θα πρέπει πρώτα να τη δει.

Technical priority (5/5): Η λειτουργία αυτή πρέπει να παρέχεται οπωσδήποτε αφού χωρίς αυτήν 
ο χρήστης θα μπορεί να βλέπει μόνο τους τίτλους των λιστών και δε θα παρέχεται στην ουσία 
καμία χρηστικότητα.

Stability (3/5): Είναι ένα χαρακτηριστικό το οποίο εξαρτάται κι από άλλες λειτουργίες άρα η 
σταθερότητα δεν είναι πολύ μεγάλη.

<ΛΑ-3>

Ο χρήστης πρέπει να μπορεί να δημιουργεί λογαριασμό.

Περιγραφή: Κατά την είσοδο του χρήστη στην κύρια σελίδα θα πρέπει να υπάρχει η επιλογή 
"Δημιουργία Λογαριασμού" έτσι ώστε ένας νέος χρήστης να μπορεί να δημιουργήσει τον προσωπικό 
του λογαριασμό. Αφού δώσει κάποια απαραίτητα στοιχεία όπως "user name", "password", "e-mail" 
κλπ, θα επιβεβαιώνει την ορθότητα των στοιχείων του και θα μεταβαίνει στην κεντρική του σελίδα.
Εκεί θα μπορεί να ξεκινήσει την δημιουργία νέων λιστών.

User priority (5/5): Η συγκεκριμένη επιλογή κρίνεται απαραίτητη διότι σε διαφορετική περίπτωση 
ο χρήστης δε θα μπορεί να εισέλθει στο σύστημα και να χρησιμοποιήσει τις δυνατότητες της εφαρμογής.

Technical priority (5/5): Η λειτουργία διασφαλίζει την είσοδο στην εφαρμογή μόνο σε πιστοποιημένους 
χρήστες έτσι ώστε να μην μπορεί ένας απλός επισκέπτης της σελίδας να χρησιμοποιεί τις δυνατότητές της.

Stability (4/5): Η διαδικασία δημιουργίας λογαριασμού πρέπει να είναι πάντα συγκεκρινμένη πράγμα 
που κάνει τη συγκεκριμένη λειτουργική απαίτηση πολύ σταθερή.

<ΛΑ-4>

Ο χρήστης πρέπει να μπορεί να ακολουθεί άλλους χρήστες.

Περιγραφή: Ο χρήστης αφού δει κάποια λίστα ή κάποιο προφίλ άλλου χρήστη που τον ενδιαφέρει 
θα πρέπει να έχει τη δυνατότητα να τον ακολουθεί. Αυτό θα πρέπει να επιτυγχάνεται με την επιλογή 
ενός κατάλληλου πλαισίου που θα υπάρχει τόσο στη σελίδα της εκάστοτε λίστας όσο και στη σελίδα 
του προφίλ του άλλου χρήστη.

User priority (4/5): Η παραπάνω επιλογή είναι σημαντικό να υπάρχει καθώς η εφαρμογή είναι ένα 
μέσο κοινωνικής δικτύωσης κι έτσι θα πρέπει οι χρήστες να μπορούν να κρατούν επαφές με αυτούς 
που έχουν κοινά ενδιαφέροντα, προτιμήσεις κλπ.

Technical priority (3/5): Είναι χρήσιμο να υπάρχει η συγκεκριμένη επιλογή αφού έτσι η εφαρμογή 
γίνεται πιο εύχρηστη και διασφαλίζει ότι η αναζήτηση συγκεκριμένων επιλογών θα είναι πιο εύκολη.

Stability (2/5): Η διαδικασία αυτή δεν χαρακτηρίζεται από μεγάλη σταθερότητα καθώς οι χρήστες 
που θα λαμβάνουν μέρος συνεχώς θα διαφοροποιούνται.

<ΛΑ-5>

Ο χρήστης πρέπει να μπορεί να βαθμολογεί λίστες.

Περιγραφή: Ο χρήστης αφού επιλέξει μια λίστα και την δει θα πρέπει να έχει τη δυνατότητα 
να τη βαθμολογήσει. Θα πρέπει να υπάρχει η συγκεκριμένη επιλογή σε πλαίσιο το οποίο θα περιέχει 
κλίμακα από 1 έως 5 (πχ αστέρια). Έτσι η κάθε λίστα θα παίρνει μια θέση στην κατάταξη με βάση 
τον μέσο όρο αυτών των βαθμολογιών.

User priority (3/5): Η συγκεκριμένη επιλογή είναι χρήσιμο να υπάρχει καθώς η βαθμολογία μιας 
λίστας παίζει ρόλο στην κατάταξή της και επομένως και στην κύρια σελίδα της εφαρμογής. Έτσι 
είναι πιο έυκολο για τους χρήστες να βρίσκουν και να βλέπουν τις πιο ενδιαφέρουσες λίστες για την 
εκάστοτε κατηγορία, καθώς και τις δικές τους προτιμήσεις.

Technical priority (3/5): Παρόλο που η παραπάνω επιλογή δεν είναι απαραίτητη για την 
λειτουργία της εφαρμογής είναι χρήσιμο να υλοποιηθεί διότι εξασφαλίζει την λειτουργικότητά της 
και εξοικονομεί χρόνο στους χρήστες αφού θα μπορούν να έχουν με σειρά κατάταξης τις 
προτιμήσεις τους.

Stability (2/5): Η διαδικασία αυτή δεν χαρακτηρίζεται από μεγάλη σταθερότητα αφού ο τρόπος βαθμολόγησης
μπορεί να διαφοροποιηθεί.

<ΛΑ-6>

Ο χρήστης πρέπει να μπορεί να ανεβάσει μια φωτογραφία προφίλ.

Περιγραφή: Κατά τη διαδικασία δημιουργίας νέου λογαριασμού από τον χρήστη θα πρέπει να 
υπάρχει η δυνατότητα επιλογής μιας φωτογραφίας ή εικόνας ώστε να μπορούν να τη βλέπουν
οι υπόλοιποι χρήστες όταν επισκέπτονται το προφίλ του. Η φωτογραφία αυτή θα προέρχεται 
από τη συσκευή που χρησιμοποιεί ο χρήστης (υπολογιστής, κινητό τηλέφωνο κλπ). Σε περίπτωση 
που ο χρήστης δεν επιθυμεί να ανεβάσει δικιά του φωτογραφία θα πρέπει να υπάρχουν κάποιες 
εικόνες προτεινόμενες από την εφαρμογή ώστε να μπορεί να επιλέξει μια από αυτές.

User priority (3/5): Ως μέσο κοινωνικής δικτύωσης η εφαρμογή είναι καλό να έχει την παραπάνω 
επιλογή καθώς διευκολύνει τους χρήστες να αναγνωρίσουν κάποιο προφίλ αφού διαφορετικά θα μπορούσαν
να το κάνουν μόνο μέσω του Username.

Technical priority (2/5): Η εφαρμογή θα ήταν πλήρως λειτουργική και χωρίς τη συγκεκριμένη 
λειτουργία ωστόσο καλό θα ήταν να υπάρχει διότι ο εμπλουτισμός των στοιχείων προφίλ του 
κάθε χρήστη κάνει πιο εύκολη την αναζήτησή του από άλλους χρήστες.

Stability (3/5): Ο τρόπος με τον οποίο ο χρήστης θα ανεβάζει φωτογραφία προφίλ θα είναι 
συγκεκριμένος ωστόσο αφήνεται ανοιχτό το ενδεχόμενο να προστεθούν και άλλα χαρακτηριστικά 
σε αυτήν την λειτουργία.

<ΛΑ-7>

Ο χρήστης πρέπει να μπορεί να επεξεργαστεί τα στοιχεία του προφίλ του.

Περιγραφή: Ο χρήστης θα έχει δώσει κάποια στοιχεία για το προφίλ του κατά τη δημιουργία 
λογαριασμού αλλά θα πρέπει να μπορεί να αλλάξει αυτά τα στοιχεία ή και να προσθέσει καινούρια.
Τα πλαίσια "Όνομα Χρήστη", "Κωδικός Πρόσβασης", "e-mail" δε θα πρέπει να μένουν κενά ενώ τα υπόλοιπα 
("Όνομα", "Επώνυμο", "τηλέφωνο", κλπ) θα είναι προαιρετικά.

User priority(5/5): Είναι πολύ σημαντικό να υπάρχει αυτή η επιλογή για το χρήστη καθώς έτσι θα μπορεί
να διορθώσει τυχόν λάθη που έγιναν κατά τη συμπλήρωση των στοιχείων ή να αποκρύψει κάποιο από
τα στοιχεία που εμφάνιζε στους άλλους χρήστες.

Technical priority(4/5): Η παραπάνω λειτουργία είναι αρκετά σημαντική καθώς ο χρήστης πρέπει 
να έχει τη δυνατότητα αναίρεσης ή διόρθωσης κάποιας επιλογής ώστε να διασφαλίζεται ότι 
είναι κάθε φορά δημοσιευμένα τα στοιχεία που επιθυμεί.

Stability (2/5): Δεν χαρακτηρίζεται απο υψηλή σταθερότητα αφού στην λειτουργία αυτή εμπεριέχονται πολλά 
χαρακτηριστικά.

<ΛΑ-8>

Ο χρήστης πρέπει να μπορεί να μετονομάσει τις λίστες.

Περιγραφή: Κάθε λίστα συνοδεύεται απαραίτητα από ένα όνομα το οποίο έχει επιλέξει ο χρήστης.
Το όνομα αυτό πρέπει να μπορεί να αλλάξει ανά πάσα στιγμή από τον χρήστη που δημοσίευσε την λίστα
και να δίνεται όποιο αυτός επιθυμεί. Για να γίνει αυτό θα πρέπει να εμφανίζεται συγκεκριμένο 
πλαίσιο όταν ο χρήστης βρίσκεται στη σελίδα της λίστας του.

User priority (3/5): Η συγκεκριμένη επιλογή έιναι καλό να υπάρχει γιατί κατά τη δημιουργία 
και ονομασία κάποιας λίστας από το χρήστη αναμένονται κάποια λάθη (πχ ορθογραφικά ή μη 
χρησιμοποίηση αντιπροσωπευτικών λέξεων) έτσι θα είναι χρήσιμο να μπορεί να τα διορθώσει 
δίνοντας ένα καινούριο όνομα.

Technical priority (2/5): Η παραπάνω λειτουργία είναι ελάσσονος σημασίας για την κυρίως 
λειτουργία της εφαρμογής και θα μπορούσε να υλοποιηθεί και χωρίς αυτήν.

Stability (2/5): Χαρακτηρίζεται από χαμηλή στεθερότητα καθώς είναι μια διαδικασία που μπορεί
να υλοποιηθεί με πολλούς τρόπους.

<ΛΑ-9>

Ο χρήστης πρέπει να μπορεί να προσθέσει σχόλια στις λίστες άλλων χρηστών.

Περιγραφή: Όταν ο χρήστης έχει εισέλθει στη σελίδα μιας λίστας θα του εμφανίζεται η 
επιλογή "Σχολιασμός Λίστας" σε πλαίσιο. Πατώντας το θα μπορεί να γράψει ένα σχόλιο 
σχετικά με τη συγκεκριμένη λίστα το οποίο θα είναι ορατό και από τους άλλους χρήστες. Ο χρήστης 
στον οποίο ανήκει η λίστα θα λαμβάνει ειδοποίηση η οποία θα αναφέρει το username του χρήστη 
που σχολίασε και το σχόλιο που έκανε.

User priority (4/5): Η λειτουργία αυτή είναι αρκετά σημαντική για το χρήστη αφού σχολιάζοντας 
τις διάφορες λίστες θα έχει τη δυνατότητα να εκθέτει την άποψή του σχετικά με αυτές και τους 
χρήστες στους οποίους ανήκουν. Επίσης αποτελεί τρόπο έμεσης επικοινωνίας και επαφής των χρηστών.

Technical priority (2/5): Η συγκεκριμένη λειτουργία δεν προσφέρει κάτι στην λειτουργικότητα 
του προγράμματος αλλά είναι επιθυμητή από τους χρήστες.

Stability (2/5): Χαρακτηρίζεται από χαμηλή στεθερότητα καθώς είναι μια διαδικασία που μπορεί
να υλοποιηθεί με πολλούς τρόπους.

<ΛΑ-10>

Ο χρήστης πρέπει να μπορεί να προσθέσει περιγραφή στις δικές του λίστες.

Περιγραφή: Αφού ο χρήστης ολοκληρώσει την επιλογή των 10 τίτλων που επιθυμεί κατά τη 
δημιουργία μιας λίστας θα πρέπει να μπορεί να συμπληρώσει (εκτός από το όνομά της) και μια 
περιγραφή σχετικά με το τι περιέχει η λίστα, σε ποιους απευθύνεται κλπ. Η περιγραφή θα 
πρέπει εμφανίζεται στους άλλους χρήστες κάτω από το όνομα της λίστας και πριν μπουν στη 
σελίδα της να τη δουν.

User priority (4/5): Είναι αρκετά σημαντικό να υπάρχει η περιγραφή σαν επιλογή από το χρήστη 
αφού έτσι θα αποφεύγει το άνοιγμα λιστών οι οποίες δεν τον ενδιαφέρουν ή αυτών που ο τίτλος 
τους δεν ξεκαθαρίζει απόλυτα περί τίνος πρόκειται.

Technical priority (3/5): Δεν είναι απαραίτητη λειτουργία της εφαρμογής ωστόσο αν υλοποιηθεί 
θα της προσθέσει λειτουργικότητα καθώς το άνοιγμα της σελίδας κάποιας λίστας θα είναι πιο 
στοχευμένο και θα εξοικονομηθεί χρόνος.

Stability (2/5): Η λειτουργία είναι χαμηλής σταθερότητας αφού ο τρόπος με τον οποίο 
κάποιος χρήστης περιγράφει τις λίστες του μπορεί να υλοποιηθεί και με διαφορετικό τρόπο από 
τον αρχικό.






<ΛΑ-11>

Ο χρήστης θα πρέπει να μπορεί να αναζητήσει άλλους χρήστες

Περιγραφή: Θα πρέπει να παρέχεται η δυνατότητα στον κάθε συνδεδεμένο χρήστη να αναζητά άλλους χρήστες βάσει του ονόματος χρήστη που αυτοί έχουν.
User priority (5/5) : Είναι ιδιαίτερα σημαντικό για όλους τους χρήστες να μπορούν να αναζητήσουν άλλους καθώς με αυτόν τον τρόπο μπορούν να δούν τις λίστες που ο καθένας από αυτούς έχει δημοσιεύσει καθώς και κάποια από τα στοιχεία τους. Ακόμη μπορούν να τους αναζητήσουν προκειμένου στη συνέχεια να συνομιλησουν με αυτούς κάτι που είναι ιδιαίτερα σημαντικό για την κοινωνική διάσταση της εφαρμογής.

Technical priority (4/5) : Η συγκεκριμένη λειτουργική απαίτηση είναι πολύ σημαντική και για την εφαρμογή αυτή καθαυτή καθώς παρέχει στους χρήστες της μεγάλη άνεση εύρεσης άλλων χρηστών κάτι που είναι ιδιαίτερα χρήσιμο. Ωστόσο η εφαρμογή θα πρέπει να είναι σε θέση να λειτουργήσει και χωρίς αυτή την απαίτηση επιτρέποντας στους χρήστες να περιηγούνται στις λίστες άλλων χωρίς αναγκαστικά να πρέπει να τις αναζητήσουν.

Stability (3/5) : Η σταθερότητα αυτής της λειτουργικής απαίτησης είναι αρκετα μεγάλη καθώς γίνεται βάσει του ονόματος χρήστη και συνεπώς δεν μπορούν να υπάρξουν μεγάλες αλλάγες σε αυτό παρά μόνο η συνεχής καταχώρηση νέων χρηστών.

<ΛΑ-12>

Ο χρήστης θα πρέπει να μπορεί να αναζητήσει λίστες.

Περιγραφή: Ο χρήστης θα έχει τη δυνατότητα να αναζητήσει τη λίστα που επιθυμεί βάσει του ονόματος της.

User priority (5/5): Οι λίστες αποτελούν τον ακρογωνιαίο λίθο της ανάπτυξης της εφαρμογής και το πεδίο ενδιαφέροντος όλων των χρηστών. Συνεπώς η αναζήτηση λιστών χρησιμοποιώντας το όνομα που τους έχει δοθεί από τους "δημιουργούς" τους είναι μια λειτουργική απαίτηση υψίστης σημασίας για τον χρήστη. Έτσι μπορεί να αναζητήσει λίστες οι οποίες είναι της αρεσκείας του και να αποφύγει την προβολή άλλων λιστών που δεν του εγείρουν το ενδιαφέρον.

Περιγραφή: Οι χρήστες θα πρέπει να ειδοποιούνται με κάποια ένδειξη όταν ένα γεγονός στην εφαρμογή υποθετικά τους ενδιαφέρει. Για 
παράδειγμα, όταν ένας φίλος τους δημοσιεύει μια καινούργια λίστα ή μία λίστα του χρήστη βαθμολογήθηκε από έναν άλλο.

User Priority: (4/5) Είναι σημαντικό ο χρήστης να χρησιμοποιεί την εφαρμογή συχνά και να τον κρατά συνέχεια σε επαφή και προσκολλημένο
σε αυτή. Οι ειδοποιήσεις το καταφέρνουν αυτό με την συνεχή υπενθύμιση για νέα συμβάντα και δημιουργούν καλύτερα το αίσθημα της άμεσης
επικοινωνίας.
 
Technical Priority: (3/5) Θεωρούμε πως είναι σημαντικό κομμάτι να ειδοποιείται ο χρήστης για τυχόν νέα γεγονότα και δεν πρέπει 
να παραλείπεται αλλά η εφαρμογή δεν παύει να είναι λειτουργική και χωρίς αυτή. 

Stability: (4/5) Δύσκολα θα αλλάξει αυτό το κομμάτι της εφαρμογής καθώς στηρίζεται στην επικοινωία των χρηστών και
δημιουργεί κοινωνικούς δεσμούς μεταξύ τους.

<ΛΑ-25>

Ο διαχειριστής θα πρέπει να μπορεί διαγράφει λίστες χρηστών. 

Περιγραφή: Ο διαχειριστής θα έχει την δυνατότητα να διαγράφει λίστες χρηστών που έχουν επισυμανθεί ότι είναι ακατάλληλες δηλαδή 
περιέχουν υλικό που προσβάλει πνευματικά δικαιώματα και ανθρώπινα δικαιώματα, περιέχει πορνογραφικό υλικό ή αποπροσανατολιστικό 
περιέχομενο και τίτλο. 

User Priority: (1/5) Για τον χρήστη δεν έχει καμία σημασία η δυνατότητα του διαχειριστή να διαγράφει το υλικό που προαναφέρθηκε.Το
σημαντικό κομμάτι είναι η ενημέρωση του χρήστη σχετικά  με τις δυνατότητες και τα χαρακτηριστικά των αναρτήσεων του έτσι ώστε να μην
προσβάλει τις συνθήκες ασφαλείας.
 
Technical Priority: (5/5) Είναι πολύ σημαντικό ο διαχειριστής να έχει το δικαίωμα διαγραφής κακόβουλων λιστών καθώς θέλουμε η εφαρμογή
να είναι προσπελάσιμη από τον χρήστη χωρίς να υπάρχει ο κίνδυνος να τον προσβάλει ή να του δημιουργηθεί αίσθηση δυσφορίας κατά την
περιήγηση σε αυτή. 

Stability: (4/5) Όπως προαναφέρθηκε αυτό το δικαιώμα του διαχειριστή είναι πολύ σημαντικό. Θα μπορούσε στο μέλλον να γίνεται αυτόματα
με κάποιου είδους τεχνητή νοημοσύνη αλλά κρίνεται πως δεν είναι τόσο αξιόπιστη όσο ο ανθρώπινος παράγοντας και η συγκριτική του
ικανότητα.


<ΛΑ-26>

Περιγραφή: Ο διαχειριστής θα έχει την δυνατότητα να διαγράφει λογαριασμούς χρηστών που έχουν κατηγορηθεί επαναλληλημένα πως έχουν

User Priority: (2/5) Ο χρήστης θέλει να νοιώθει ασφάλεια καθώς περιηγείται στην εφαρμογή. Η γνώση πως υπάρχει κάποιος έλεγχος και
ουσιαστικά η τιμωρία με διαγραφή του λογαριασμού αποτρέπει κινήσεις που εναντιώνοτναι στην πολιτική της εφαρμογής. 

 
Technical Priority: (5/5) Είναι πολύ σημαντικό ο διαχειριστής να έχει το δικαίωμα διαγραφής κακόβουλων λογαριασμών καθώς θέλουμε η
εφαρμογή να είναι προσπελάσιμη από τον χρήστη χωρίς να υπάρχει ο κίνδυνος να τον προσβάλει ή να του δημιουργηθεί αίσθηση δυσφορίας κατά
την περιήγηση σε αυτή. 

Stability: (4/5) Όπως προαναφέρθηκε (ΛΑ-25) αυτό το δικαιώμα του διαχειριστή είναι πολύ σημαντικό. Θα μπορούσε στο μέλλον να γίνεται
αυτόματα με κάποιου είδους τεχνητή νοημοσύνη αλλά κρίνεται πως δεν είναι τόσο αξιόπιστη όσο ο ανθρώπινος παράγοντας και η συγκριτική του
ικανότητα.
